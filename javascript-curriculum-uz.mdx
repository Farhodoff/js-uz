# JavaScript - Loyihaga Asoslangan O'quv Kursi

Bu kurs JavaScript'ni amaliy loyiha fikri bilan o'rgatadi. Har bir dars avval tushunchani beradi, keyin kichik topshiriq bilan uni darhol qo'llashga majbur qiladi. Ketma-ketlik oddiydan murakkabga o'tadi: avval kodni konsolda tushunish, keyin ma'lumotlar, funksiya va boshqaruv oqimi, undan keyin DOM, asinxronlik va yakuniy mini-loyiha.

## 1. JavaScript bilan tanishuv

### Nazariy qism
JavaScript brauzer ichida ishlaydigan, lekin endi server tomonda ham keng qo'llaniladigan dasturlash tili. U sahifaga hayot beradi: tugmalarni bosish, forma tekshirish, animatsiya, ma'lumot olish va ekranni qayta chizish kabi ishlar JS orqali amalga oshadi. Boshlanishida eng muhim odat - kodni konsolda sinash. `console.log()` sizga qiymatlarni ko'rishga, xatolarni erta topishga va fikrni mayda qadamlar bilan tekshirishga yordam beradi.

```javascript
console.log("Salom, JavaScript!");
console.log(2 + 3);
```

Kurs davomida siz ko'pincha uchta savolga javob topasiz: qaysi qiymat bilan ishlayapman, u qayerda saqlangan, va nima uchun aynan shu natija chiqdi? Javobni aynan shunday fikrlash uslubi beradi. Brauzer, DOM, konsol va skript fayli o'rtasidagi bog'lanishni erta tushunsangiz, keyingi darslar ancha yengil ko'rinadi.

### Amaliy vazifa
1. Konsolda ism, shahar va sevimli kasbingizni `console.log()` bilan chiqaring.
2. `sum` nomli funksiya yozmasdan, oddiy matematik ifoda bilan 3 ta sonni qo'shib ko'ring.
3. HTML sahifa ichida `<script>` orqali JS ulanganini tekshiring.

### Tavsiyalar va hints
- Avval natijani o'zingiz bashorat qiling, keyin kodni yozing.
- Har bir qatorni alohida sinab ko'ring.
- Xatoni ko'rganingizda qo'rqmang, u o'qituvchi vazifasini bajaradi.

```javascript
console.log("Ism: Farhod");
console.log("Shahar: Toshkent");
console.log(10 + 20 + 30);
```

### Quiz
1. JavaScript nimaga kerak?
2. `console.log()` nima qiladi?
3. Nega kodni konsolda tekshirish foydali?

## 2. O'zgaruvchilar va ma'lumot turlari

### Nazariy qism
O'zgaruvchi - bu qiymat saqlaydigan konteyner. JavaScript'da `let`, `const` va tarixiy `var` bor. Dars boshida asosan `let` va `const` bilan ishlaymiz: `let` qayta qiymat berish mumkin bo'lgan holatlar uchun, `const` esa o'zgarmas havola uchun ishlatiladi. `const` obyekt yoki massiv ichidagi elementlarni o'zgartirishga mutlaqo to'siq bo'lmaydi; u faqat o'zgaruvchining o'ziga boshqa qiymat berishni cheklaydi.

Ma'lumot turlari ham poydevor hisoblanadi. `string`, `number`, `boolean`, `undefined`, `null`, `object`, `symbol`, `bigint` bilan tanishish keyingi darslarda xatolarni kamaytiradi. Ayniqsa `typeof` operatori sizga kelgan qiymatning turini tekshirishda juda kerak bo'ladi.

```javascript
let ism = "Ali";
const yosh = 21;
let faol = true;

console.log(typeof ism);
console.log(typeof yosh);
console.log(typeof faol);
```

### Amaliy vazifa
1. `name`, `age`, `isStudent` o'zgaruvchilarini yarating.
2. Har birining turini konsolga chiqaring.
3. `const` bilan massiv yarating va ichiga 3 ta sevimli narsangizni yozing.

### Tavsiyalar va hints
- `null` va `undefined` farqini albatta yodda tuting.
- `typeof null` natijasi tarixiy xatolik sabab `object` qaytarishini bilib qo'ying.
- Qiymatlar ustida hisob yoki matn biriktirishdan oldin turini tekshiring.

```javascript
const sevimliRanglar = ["ko'k", "yashil", "oq"];
console.log(typeof sevimliRanglar);
console.log(sevimliRanglar[0]);
```

### Quiz
1. `let` va `const` qanday farqlanadi?
2. `typeof` nima uchun kerak?
3. `null` va `undefined` o'rtasidagi farq nima?

## 3. Operatorlar va shartlar

### Nazariy qism
Operatorlar qiymatlar ustida hisob va taqqoslash amalga oshiradi. Matematik operatorlar `+`, `-`, `*`, `/`, `%` va `**` tez-tez ishlatiladi. Taqqoslash operatorlari `===`, `!==`, `>`, `<`, `>=`, `<=` esa qaror qabul qilishning asosi hisoblanadi. Shartli tarmoqlanish uchun `if`, `else if`, `else` kerak bo'ladi.

Bu bosqichda eng muhim odat - har bir shartni oddiy gapga tarjima qilish. Masalan: "Agar foydalanuvchi 18 yoshdan katta bo'lsa, kirishga ruxsat ber." Shu gapni yozma mantqqa aylantirsangiz, kod ancha tabiiy chiqadi.

```javascript
const yosh = 19;

if (yosh >= 18) {
  console.log("Ruxsat beriladi");
} else {
  console.log("Ruxsat berilmaydi");
}
```

`switch` ko'p holatli tanlashda qulay, ayniqsa hafta kunlari, buyruqlar yoki menyu variantlarida. Taqqoslashda `===` ni afzal ko'ring, chunki u tur va qiymatni birga tekshiradi.

### Amaliy vazifa
1. Berilgan son juft yoki toqligini aniqlang.
2. 1 dan 7 gacha bo'lgan raqamga qarab hafta kunini chiqaring.
3. Ballga qarab baho yozing: 90+ A, 70+ B, 50+ C, aks holda F.

### Tavsiyalar va hints
- Avval qog'ozda shartlar ro'yxatini yozing.
- Har bir `if` faqat bitta vazifani bajarsin.
- `switch` faqat aniq qiymatlar uchun qulay.

```javascript
const son = 12;
if (son % 2 === 0) {
  console.log("Juft son");
} else {
  console.log("Toq son");
}
```

### Quiz
1. `===` va `==` o'rtasidagi farq nima?
2. Qachon `switch` dan foydalanish qulay?
3. `%` operatori nima qiladi?

## 4. Sikliklar va takrorlash

### Nazariy qism
Sikliklar bir xil ishni ko'p marta takrorlash uchun ishlatiladi. `for` odatda takrorlash soni oldindan ma'lum bo'lganda yaxshi ishlaydi. `while` esa shart bajarilib turguncha davom etadi. `break` siklni to'xtatadi, `continue` esa joriy aylanishni tashlab keyingisiga o'tadi.

Takrorlashni o'rganishda eng yaxshi yo'l - kichik masalalar. Masalan, 1 dan 10 gacha sonlarni chiqarish, faqat juft sonlarni olish yoki massiv ichidagi elementlarni tekshirish. Shu kabi mashqlar nafaqat sintaksisni, balki algoritmik fikrlashni ham o'stiradi.

```javascript
for (let i = 1; i <= 5; i++) {
  console.log(i);
}

let count = 0;
while (count < 3) {
  console.log("Hisob: " + count);
  count++;
}
```

`for...of` massiv elementlari bilan ishlashga qulay, `for...in` esa obyekt kalitlari uchun ishlatiladi. Ularni aralashtirib yubormang.

### Amaliy vazifa
1. 1 dan 20 gacha sonlarni chiqaring.
2. Faqat juft sonlarni alohida ko'rsating.
3. Massivdagi 5 ta mevaning har birini konsolga yozing.

### Tavsiyalar va hints
- Sikl ichida qiymatlar qanday o'zgarishini kuzating.
- `break` va `continue` ni kichik misolda sinab ko'ring.
- Massiv bilan ishlaganda `for...of` dan boshlang.

```javascript
const mevalar = ["olma", "nok", "banan"];
for (const meva of mevalar) {
  console.log(meva);
}
```

### Quiz
1. `for` va `while` farqi nima?
2. `break` nima qiladi?
3. `for...of` qachon ishlatiladi?

## 5. Funksiyalar va qayta foydalanish

### Nazariy qism
Funksiya - qayta ishlatiladigan kod bo'lagi. Katta loyihada bir ishni takror yozish o'rniga funksiya yozish kodni toza, qisqa va boshqariladigan qiladi. Funksiyalar parametr qabul qiladi, natija qaytaradi va tashqi dunyo bilan muloqot qiladi. Bu darsda siz oddiy funksiya, arrow function, default parametr va `return` tushunchalarini mustahkamlaysiz.

```javascript
function salomBer(ism) {
  return `Salom, ${ism}`;
}

const kvadrat = (son) => son * son;

console.log(salomBer("Farhod"));
console.log(kvadrat(4));
```

Funksiya ichida hisoblangan qiymatni keyin yana ishlatish uchun `return` kerak. Agar faqat `console.log` qilsangiz, siz qiymatni ko'rasiz, lekin boshqa joyda ishlata olmaysiz. Shu sababli boshlanishda ko'p masalalarda `return` ni unutmaslik juda muhim.

### Amaliy vazifa
1. Ikki sonni qo'shadigan funksiya yozing.
2. Berilgan sonning kubini qaytaradigan arrow function yozing.
3. Ism va familiyani birlashtiradigan funksiya yozing.

### Tavsiyalar va hints
- Funksiyani avval nomlang, keyin parametrlarini aniqlang.
- Kichik masalalarni bitta funksiyaga yuklang.
- Funksiya natijasini `console.log` bilan alohida tekshirib boring.

```javascript
function sum(a, b) {
  return a + b;
}

const kub = (n) => n ** 3;
console.log(sum(2, 3));
console.log(kub(5));
```

### Quiz
1. Funksiya nima uchun kerak?
2. `return` va `console.log` farqi nima?
3. Arrow function qachon qulay?

## 6. Massivlar bilan ishlash

### Nazariy qism
Massiv - bir nechta qiymatni tartibli saqlash uchun ishlatiladi. JavaScript'da massivlar bilan ishlash juda muhim, chunki ko'p loyihalarda ro'yxatlar, kartalar, mahsulotlar va foydalanuvchilar to'plami aynan massiv ko'rinishida keladi. Siz `push`, `pop`, `map`, `filter`, `find`, `slice`, `splice` kabi metodlarni bilishingiz kerak.

```javascript
const sonlar = [1, 2, 3, 4];
sonlar.push(5);

const juftlar = sonlar.filter((son) => son % 2 === 0);
const kvadratlar = sonlar.map((son) => son * son);

console.log(juftlar);
console.log(kvadratlar);
```

Massivni o'zgartirish va undan yangi massiv yaratish o'rtasidagi farqni tushunish juda foydali. `map` ko'pincha yangi natija kerak bo'lganda ishlatiladi, `filter` esa shartga mos elementlarni qoldiradi. Lo'yiha kodida bu metodlar eng ko'p uchraydi.

### Amaliy vazifa
1. 5 ta raqamdan iborat massiv yarating.
2. Har bir sonni 2 ga ko'paytirib yangi massiv oling.
3. Faqat 10 dan katta sonlarni alohida massivga ajrating.

### Tavsiyalar va hints
- Bir xil vazifa uchun bir necha metodni sinab ko'ring.
- Massivni konsolda ko'rib borish odatini qiling.
- `map` va `filter` ni aralashtirib yubormang.

```javascript
const narxlar = [100, 250, 50, 400];
const qimmat = narxlar.filter((narx) => narx > 100);
console.log(qimmat);
```

### Quiz
1. `map` va `filter` farqi nima?
2. Massivga yangi element qo'shish uchun qaysi metod ishlatiladi?
3. Nima uchun massivlar loyihalarda ko'p ishlatiladi?

## 8. Destructuring va spread/rest

### Nazariy qism
Destructuring - obyekt yoki massiv ichidan qiymatlarni qisqa usulda ajratib olish. Bu kodni ixcham va tushunarli qiladi. Spread operator esa mavjud massiv yoki obyektni kengaytiradi, nusxalaydi yoki yangi tuzilmaga joylaydi. Rest parametr va rest destructuring esa ortiqcha elementlarni bir joyga to'playdi.

```javascript
const user = { name: "Ali", age: 25, city: "Samarkand" };
const { name, age } = user;

const nums = [1, 2, 3];
const moreNums = [...nums, 4, 5];

console.log(name, age);
console.log(moreNums);
```

Bu bilimlar loyiha kodini soddalashtiradi. Masalan, formadan kelgan obyektning faqat kerakli qismini olish, props uzatish yoki massivga yangi item qo'shish kabi vaziyatlarda destructuring juda foydali. Katta loyihada har safar `user.name` deb yozish o'rniga kerakli property'ni oldindan ajratib olasiz.

### Amaliy vazifa
1. Obyektdan 2 ta qiymatni destructuring bilan oling.
2. Massivga spread orqali 2 ta yangi element qo'shing.
3. Funksiya yozib, istalgan sonli argumentlarni rest orqali yig'ing.

### Tavsiyalar va hints
- Har doim asl obyekt o'zgaryaptimi yoki yangi nusxa yaratyaptimi, tekshiring.
- Spread va rest yozuvi tashqi ko'rinishda o'xshash, lekin maqsadi boshqacha.
- Destructuring'da default qiymat berishni ham sinab ko'ring.

```javascript
const numbers = [10, 20, 30];
const [first, second] = numbers;
function sum(...args) {
  return args.reduce((total, item) => total + item, 0);
}
```

### Quiz

3. Rest parametr nimani yig'adi?

## 9. Arrow functions, callback va this

### Nazariy qism
Arrow function qisqaroq yozuv beradi, lekin u `this` ni o'zining alohida konteksti sifatida yaratmaydi. Shu sababli obyekt metodlari, callbacklar va yuqori darajali funksiyalarni yozishda uning xususiyatini bilish kerak. Callback - bu boshqa funksiyaga argument sifatida beriladigan funksiya. `map`, `filter`, `setTimeout` va hodisalar bilan ishlashda callbacklar juda ko'p uchraydi.

```javascript
const greet = (name) => `Salom, ${name}`;

function bajar(callback) {
  console.log("Boshlanish");
  callback();
}

bajar(() => console.log("Tugadi"));
```

`this` esa qayerdan chaqirilganiga qarab qiymat oladi. Bu mavzu ko'pincha chalkashlik keltiradi, shuning uchun obyekt metodlari, `call`, `apply`, `bind` va oddiy funksiya bilan sinash juda foydali. Hozircha eng muhim narsa: arrow function va oddiy function o'rtasida farq borligini amalda ko'rish.

### Amaliy vazifa
1. 3 ta sonni ko'paytiradigan arrow function yozing.
2. Massivdagi elementlarni `map` bilan katta harfga aylantiring.
3. Obyekt ichida `this` bilan ismni chiqaring.

### Tavsiyalar va hints
- `this` muammosini topish uchun obyekt ichida va tashqarisida sinab ko'ring.
- Arrow function'ni hamma joyda ishlatishga shoshilmang.
- Callback natijasini konsolda alohida ko'ring.

```javascript
const user = {
  name: "Farhod",
  greet() {
    console.log(`Salom, ${this.name}`);
  }
};
user.greet();
```

### Quiz

## 10. DOM bilan ishlash

### Nazariy qism
DOM - bu HTML hujjatining daraxt ko'rinishdagi modeli. JavaScript DOM orqali elementlarni topadi, matnini o'zgartiradi, class qo'shadi, tugma bosilganda sahifa holatini yangilaydi. Frontend dasturchisi uchun DOM - bu foydalanuvchi bilan bevosita muloqot qiladigan maydon.

```javascript
const title = document.querySelector("h1");
title.textContent = "Yangi sarlavha";
title.classList.add("active");
```

DOM bilan ishlashni o'rganishda avval elementni topish, keyin uni o'zgartirish, undan so'ng hodisa qo'shish tartibi yaxshi ishlaydi. `querySelector`, `getElementById`, `classList`, `textContent`, `innerHTML` kabi metodlar eng zarurlari hisoblanadi. O'zgartirishlar faqat kodda emas, sahifada ko'rinishi bilan ham tekshiriladi.

### Amaliy vazifa
1. Sahifadagi bir sarlavhani topib, matnini o'zgartiring.
2. Tugmaga bosilganda fon rangini almashtiring.
3. Ro'yxatga yangi element qo'shishni sinab ko'ring.

### Tavsiyalar va hints
- Har safar element topildi-mi, tekshirib boring.
- `console.log(element)` orqali DOM obyektini ko'ring.
- `innerHTML` ni ehtiyotkorlik bilan ishlating.

```javascript
const button = document.querySelector("button");
button.addEventListener("click", () => {
  document.body.classList.toggle("dark");
});
```

### Quiz
1. DOM nima?
2. `querySelector` nima qiladi?
3. `textContent` va `innerHTML` farqi nima?

## 11. Hodisalar va foydalanuvchi bilan aloqa

### Nazariy qism
Event yoki hodisa - bu foydalanuvchi yoki brauzer tomonidan yuzaga keladigan signal. Click, input, submit, keydown, change kabi hodisalar yordamida interaktiv ilovalar quriladi. `addEventListener` hodisani tinglashning to'g'ri usuli hisoblanadi. Bu darsda siz hodisa obyektini, `preventDefault()` ni va forma bilan ishlashni tushunasiz.

```javascript
const form = document.querySelector("form");
form.addEventListener("submit", (event) => {
  event.preventDefault();
  console.log("Forma yuborildi");
});
```

Hodisa bilan ishlaganda asosiy maqsad - foydalanuvchi harakatiga javob berish. Masalan, input yozilganda ro'yxat filtrlash, tugma bosilganda element qo'shish, forma yuborilganda tekshirish. Bu amallarni mayda qismlarga bo'lish yaxshi natija beradi.

### Amaliy vazifa
1. Tugma bosilganda hisoblagichni 1 ga oshiring.
2. Inputga yozilgan matnni ekranga chiqaring.
3. Formani yuborganda sahifa qayta yuklanmasin.

### Tavsiyalar va hints
- Hodisa ichida `event` obyektini ko'rib chiqing.
- Bir elementga bir nechta event qo'shib tekshiring.
- `preventDefault()` ni faqat kerak bo'lganda ishlating.

```javascript
const input = document.querySelector("input");
const output = document.querySelector(".output");
input.addEventListener("input", (event) => {
  output.textContent = event.target.value;
});
```

### Quiz
1. `addEventListener` nima uchun kerak?
2. `preventDefault()` nima qiladi?
3. Qaysi hodisalar interaktiv sahifada ko'p uchraydi?

## 12. Promise va asinxron fikrlash

### Nazariy qism
Promise - kelajakdagi natijani ifodalovchi obyekt. Asinxron kodda natija darhol kelmasligi mumkin, shuning uchun promise yordamida kutish, muvaffaqiyat yoki xatolik holatlarini boshqaramiz. `pending`, `fulfilled`, `rejected` holatlari asosiy tushunchalardir. `then`, `catch`, `finally` orqali oqimni boshqarasiz.

```javascript
const promise = new Promise((resolve) => {
  setTimeout(() => resolve("Tayyor"), 1000);
});

promise.then((result) => console.log(result));
```

Promise'lar real loyihada API so'rovi, fayl o'qish yoki vaqtinchalik kechikishlarda juda ko'p uchraydi. Ularni tushunish keyingi `async/await` darsining ham poydevori hisoblanadi. Eng muhim odat - chaining qilishdan qo'rqmaslik va xatolarni `catch` bilan ushlash.

### Amaliy vazifa
1. 2 soniyadan keyin matn qaytaradigan promise yozing.
2. Promise muvaffaqiyatsiz bo'lsa xato matnini chiqaring.
3. `then` va `catch` ni birga ishlating.

### Tavsiyalar va hints
- Promise ichida `resolve` va `reject` ni sinab ko'ring.
- `setTimeout` asinxronlikni ko'rish uchun yaxshi misol.
- Xato bo'lsa, avval promise konstruktori ichini tekshiring.

```javascript
const p = new Promise((resolve, reject) => {
  const success = true;
  if (success) resolve("OK");
  else reject("Xato");
});
```

### Quiz
1. Promise nima?
2. `then` va `catch` vazifasi nima?
3. `pending` holati nimani bildiradi?

## 13. Async/await va API bilan ishlash

### Nazariy qism
`async/await` promise ustiga yozilgan yanada o'qilishi oson sintaksisdir. U kodni ketma-ket o'qishga o'xshatadi, lekin aslida asinxron ishlaydi. `async` funksiya har doim promise qaytaradi, `await` esa promise natijasini kutadi. `try/catch` bilan xatoni boshqarish juda muhim.

```javascript
async function dataOl() {
  try {
    const response = await fetch("https://api.example.com/users");
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}
```

API bilan ishlaganda response status, network xatosi va JSON parsing muammolarini alohida tushunish kerak. `Promise.all` bir nechta so'rovni parallel yuboradi. Bu darsda foydalanuvchilar yoki postlar ro'yxatini olib keladigan kichik misol yaxshi amaliyot bo'ladi.

### Amaliy vazifa
1. `async` funksiya yozing va 1 soniya kuting.
2. `Promise.all` bilan 2 ta so'rovni parallel bajaring.
3. Xato bo'lsa `catch` ichida xabar chiqaring.

### Tavsiyalar va hints
- `await` faqat `async` ichida ishlashini esda tuting.
- API natijasini ishlatishdan oldin JSON ga aylantiring.
- `try/catch` ni o'rab ishlash odatini qiling.

```javascript
async function kutish() {
  await new Promise((resolve) => setTimeout(resolve, 1000));
  return "Bajarildi!";
}
```

### Quiz
1. `async` nima qiladi?
2. `await` qachon ishlatiladi?
3. Nega `try/catch` kerak?

## 14. LocalStorage, error handling va modules

### Nazariy qism
Loyiha ma'lumotini saqlash uchun `localStorage` juda foydali. U brauzer yopilib-ochilganda ham qoladigan oddiy kalit-qiymat ombori. Odatda massiv yoki obyektni saqlashdan oldin `JSON.stringify`, o'qiganda esa `JSON.parse` ishlatiladi. Bu darsda xato boshqarish ham, modulga ajratish ham birga ko'rib chiqiladi.

```javascript
const user = { name: "Ali", age: 25 };
localStorage.setItem("user", JSON.stringify(user));

const savedUser = JSON.parse(localStorage.getItem("user"));
console.log(savedUser);
```

Xato boshqarishda `try/catch/finally` foydali. Masalan, JSON parse qilish, noto'g'ri forma ma'lumotlari yoki kutilmagan qiymatlar bilan ishlashda bu bloklar kerak bo'ladi. `throw` orqali o'zingiz xatolik chiqarishingiz ham mumkin. Modullar esa kodni alohida fayllarga bo'lishga yordam beradi: eksport qiling, kerak bo'lgan joyda import qiling.

### Amaliy vazifa
1. Obyektni localStorage ga saqlang va qayta o'qing.
2. Noto'g'ri JSON ni `try/catch` bilan ushlang.
3. Ikki funksiyani alohida modulga ajrating.

### Tavsiyalar va hints
- Saqlashdan oldin string, o'qiganda obyekt kerakligini unutmang.
- `finally` har qanday holatda ishlashini tekshirib ko'ring.
- Modul nomlarini ma'noli qo'ying.

```javascript
try {
  const data = JSON.parse("{xato json}");
} catch (error) {
  console.log("JSON xato");
}
```

### Quiz
1. localStorage nima uchun kerak?
2. Nega `JSON.stringify` ishlatiladi?
3. `throw` va `catch` farqi nima?

## 15. Yakuniy loyiha: Todo List

### Nazariy qism
Yakuniy bosqichda siz kichik, lekin real mahsulotga o'xshash todo list quraysiz. Bu loyiha oldingi barcha darslarni birlashtiradi: massivlar, obyektlar, funksiyalar, DOM, eventlar, localStorage va xato boshqarish. Eng yaxshi yondashuv - funksiyalarni kichik bloklarga bo'lish: qo'shish, chizish, o'chirish, bajarilgan deb belgilash, saqlash va yuklash.

```javascript
let todos = [];

function addTodo(text) {
  todos.push({
    id: Date.now(),
    text,
    completed: false
  });
}
```

Loyiha bosqichida kodni darhol mukammal qilishga urinmang. Avval minimal ishlaydigan variantni yarating, keyin xususiyat qo'shing. Masalan, avval faqat qo'shish va chiqarish, so'ng o'chirish, keyin `completed` holati, undan keyin localStorage. Shu tartib sizni chalkashlikdan saqlaydi.

### Amaliy vazifa
1. Todo qo'shish funksiyasini yozing.
2. Todo'ni o'chirish va `completed` holatini almashtirishni qo'shing.
3. Todo ro'yxatini localStorage ga saqlang.
4. Sahifa yangilanganda ro'yxat qayta yuklansin.

### Tavsiyalar va hints
- Har funksiyani alohida sinab ko'ring.
- `renderTodos()` bilan ekrandagi ro'yxatni boshqaring.
- `map`, `filter` va `JSON.stringify/parse` ni birga ishlating.

```javascript
function toggleTodo(id) {
  todos = todos.map((todo) =>
    todo.id === id ? { ...todo, completed: !todo.completed } : todo
  );
}
```

2. Nega ro'yxatni funksiyalarga bo'lish kerak?
3. localStorage qachon foydali bo'ladi?
